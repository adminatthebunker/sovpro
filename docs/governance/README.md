# Governance

Public-facing policy documents. Per-feature: scope, what is and isn't permitted, complaint and correction workflows, retention rules, legal posture.

These are policy not architecture. The "Always-on" governance items in [`docs/timeline.md`](../timeline.md) — takedown / correction policy, DSAR workflow, AI-report disclaimer copy — land here as they're written. `docs/plans/` describes how features get built; this directory describes the rules under which they operate.
