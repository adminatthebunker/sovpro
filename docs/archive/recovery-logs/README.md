# Recovery Logs

Point-in-time incident-recovery notes. Snapshots of system state during specific reboots, outages, or one-off historical-backfill efforts between 2026-04-16 and 2026-04-23.

These are **not operational runbooks** — operational runbooks live in [`docs/runbooks/`](../../runbooks/) and are evergreen procedures. Files here are dated artifacts: the commands they list may reference paths, container IDs, or schema states that no longer exist.

If a file here documents a procedure that should be evergreen (the billing-rail handoff is a candidate), extract the procedure into `docs/runbooks/` and leave the dated narrative behind.
